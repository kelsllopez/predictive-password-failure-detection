> Proyecto educativo enfocado en el uso de análisis predictivo para identificar futuras vulnerabilidades relacionadas con las tendencias de creación de contraseñas dentro de una organización. El objetivo es anticipar riesgos antes de que sean explotados y fortalecer la estrategia de autenticación mediante recomendaciones preventivas.

# 📖 Descripción

Las contraseñas siguen siendo uno de los mecanismos de autenticación más utilizados, pero también uno de los principales objetivos de los ciberdelincuentes. Aunque muchas organizaciones implementan políticas de complejidad, las tendencias de creación de contraseñas pueden revelar patrones que aumentan el riesgo de futuras brechas de seguridad.

Este proyecto propone un enfoque predictivo para analizar dichas tendencias, identificar comportamientos inseguros y generar recomendaciones estratégicas que permitan anticipar vulnerabilidades antes de que sean aprovechadas por un atacante.

El objetivo es pasar de una estrategia reactiva a una estrategia preventiva, utilizando análisis de datos para fortalecer la seguridad de las credenciales.

# 📝 Prompt

> Analiza las tendencias actuales de creación de contraseñas dentro de una organización para identificar posibles vulnerabilidades futuras. Evalúa patrones de longitud, complejidad, reutilización, uso de palabras comunes e información personal. Con base en los resultados, predice los riesgos que podrían facilitar ataques como fuerza bruta, password spraying o credential stuffing, y genera recomendaciones estratégicas para fortalecer la política de contraseñas y reducir la probabilidad de futuras brechas de seguridad.

# 🔍 Factores Analizados

El análisis considera diversos aspectos relacionados con la creación de contraseñas.

## 📏 Longitud

- Contraseñas menores a 12 caracteres.
- Longitud promedio.
- Evolución de la longitud en el tiempo.

---

## 🔤 Complejidad

Se analiza el uso de:

- Letras mayúsculas.
- Letras minúsculas.
- Números.
- Caracteres especiales.

---

## 🔄 Reutilización

Se identifican:

- Contraseñas repetidas.
- Patrones similares entre usuarios.
- Reutilización en diferentes sistemas.

---

## 📚 Palabras Comunes

Se detecta el uso de:

- Nombres propios.
- Fechas.
- Palabras del diccionario.
- Nombre de la empresa.
- Variaciones simples de contraseñas conocidas.

---

## 👤 Información Personal

El sistema identifica posibles coincidencias con:

- Nombre del usuario.
- Apellidos.
- Fechas importantes.
- Información pública.

# 🚨 Riesgos Potenciales

A partir de las tendencias observadas se pueden prever riesgos como:

- Ataques de fuerza bruta.
- Password Spraying.
- Credential Stuffing.
- Ataques de diccionario.
- Compromiso de cuentas privilegiadas.
- Accesos no autorizados.
- Escalamiento de privilegios.

---

# 📊 Clasificación del Riesgo

| Nivel | Descripción |
|--------|-------------|
| 🟢 Bajo | Tendencias seguras y consistentes. |
| 🟡 Medio | Existen patrones que requieren revisión. |
| 🟠 Alto | Se detectan múltiples debilidades. |
| 🔴 Crítico | Alta probabilidad de explotación futura. |

# 💡 Recomendaciones Estratégicas

Con base en el análisis, se recomienda:

- Aumentar la longitud mínima de las contraseñas.
- Implementar frases de contraseña (Passphrases).
- Activar autenticación multifactor (MFA).
- Impedir la reutilización de contraseñas.
- Bloquear el uso de contraseñas comunes.
- Actualizar periódicamente la política de contraseñas.
- Capacitar a los usuarios sobre buenas prácticas.
